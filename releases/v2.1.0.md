# AWS Incident Response Playbook v2.1.0

**Release name:** Automation Framework

## Highlights

- Introduces the Phase 3 response-automation foundation.
- Adds five modular, dry-run-first Lambda actions.
- Adds shared Python helpers, unit tests, sample events, IAM examples, and Terraform deployment.
- Adds CI checks for Python automation and Terraform configuration.
- Intentionally leaves automatic detection triggers and orchestration disabled for safety.

## Added

- `automation/` framework and documentation.
- EC2 metadata, isolation, and EBS snapshot functions.
- IAM access-key disablement and SNS notification functions.
- Least-privilege policy examples and permissions matrix.
- Terraform lab deployment and GitHub Actions validation.

## Safety note

Deploy in a dedicated lab account first. Mutating events default to dry run, but responders remain responsible for authorization, evidence preservation, impact analysis, and rollback planning.
