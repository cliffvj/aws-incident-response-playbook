# AWS Incident Response Playbook v1.2.0

**Release name:** Visual Documentation

## Overview

Version 1.2.0 improves the operational readability of the playbook without changing the core response objectives. Every numbered scenario now begins with an incident snapshot and includes a GitHub-rendered Mermaid response flow. Reusable diagram sources are stored in `diagrams/` for future documentation and presentation workflows.

## Added

- Incident snapshot tables for all 20 runbooks.
- Mermaid response-flow diagrams for all 20 runbooks.
- Four architecture-level Mermaid diagrams.
- Diagram source catalog and conventions.
- Previous, next, and documentation-index links in every scenario.

## Updated

- Main README navigation, repository map, and lab-safety guidance.
- Documentation index with visual-documentation links.
- Changelog with the visual-documentation milestone.

## Safety

The diagrams are operational patterns, not authorization to modify production resources. Validate account, Region, resource identity, evidence requirements, approvals, and rollback controls before any write action.

## Coming next

Version 1.3.0 will add MITRE ATT&CK, NIST incident-response, and AWS Well-Architected mappings while preserving the existing runbook structure.
