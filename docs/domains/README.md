# Response Domain Indexes

Use these indexes when the exact scenario is unclear or an incident spans multiple services. Each page groups related runbooks, supporting procedures, and likely evidence sources.

| Domain | Use when |
|---|---|
| [Compute and workloads](compute-workloads.md) | EC2, Auto Scaling, Systems Manager, EBS, workload isolation, or forensic collection is involved. |
| [Identity and accounts](identity-account.md) | Access keys, IAM users or roles, root, sessions, or unauthorized API activity is involved. |
| [Data, storage, and databases](data-storage-databases.md) | S3, RDS, data exposure, exfiltration, or ransomware is involved. |
| [Detection, audit, and compliance](detection-audit-compliance.md) | CloudTrail, CloudWatch, Config, Athena, evidence discovery, or control drift is involved. |
| [Network, automation, and recovery](network-automation-recovery.md) | Security groups, Lambda, Step Functions, containment workflows, or trusted restoration is involved. |
